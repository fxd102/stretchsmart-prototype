DESIGN NOTES

V1 vs V2: Today's Focus

Comparison to help finalize the initial design

V1 — The Original

  • One exercise per day, picked automatically by date
  • Simple timer ring with the exercise's built-in duration
  • "Let's Go" → timer counts down → fun done message ("Crushed it!", "Easy money.")
  • Motivational quote below the timer
  • Swap panel with position filters (seated / standing / floor) to pick a different exercise
  • Session counter ("Completed today: 3")
  • Pulls exercises from the API

V2 — The Redesign

  • "Nudge card" model — multiple nudges per day (shows "Nudge 1 of 4")
  • Narrative hooks that change per nudge ("Your shoulders carried a lot yesterday. 3 minutes?" instead of generic prompts)
  • Duration slider: you choose 1–5 minutes per session (default 3)
  • "Days you've moved" counter at the top — only goes up, never resets
  • Each exercise shows a progression hint that adapts ("Last week you held 10 seconds. Try 15.")
  • Research insight with real citation after each exercise
  • Snooze button (15 / 30 / 45 min) — always available
  • Swap button — but only on the first nudge of the day
  • Post-exercise feedback (Easy / Challenging / Couldn't complete) that drives future difficulty
  • Exercises are hardcoded (not pulling from the API yet)

V1 Pros

  • Simpler, less overwhelming for first-time users
  • Connected to the exercise API (easier to expand the library)
  • Position filters (seated / standing / floor) are genuinely useful — v2 doesn't have those yet
  • "Again" button lets you repeat the same exercise multiple times in one sitting

V1 Cons

  • No personalization or adaptation
  • Timer duration is fixed per exercise — no user control
  • Generic motivational quotes don't land as well as v2's narrative hooks
  • No feedback loop — the app never learns what's easy or hard for you
  • No snooze — it's now-or-never

V2 Pros

  • The nudge model matches how the app will actually work (notifications throughout the day)
  • Narrative hooks make it feel personal, not robotic
  • Feedback → adaptation is the killer feature — the app gets smarter over time
  • Duration slider gives the user control without complexity
  • Anti-metric ("days moved") is motivating without being punishing
  • Snooze respects the user's real life
  • Research insights give each session a "learn something" reward

V2 Cons

  • More complex — more to build, more to break
  • Exercises are hardcoded, not pulling from the API yet
  • No position filters for swapping (v1 had seated / standing / floor, great for "I'm at my desk")
  • The design notes panel at the bottom is dev-facing, not user-facing (minor, easy to hide)

My Suggestions

  1. Go with v2 as the base. The feedback loop and nudge model are what make StretchSmart different from a generic stretch timer.
  2. Port v1's position filters into v2's swap panel. "I'm seated right now" is a real use case.
  3. Connect v2 to the exercise API instead of hardcoded data.
  4. Keep v1's "Again" button concept. After completing a nudge, let the user do another round if they want, instead of just showing "done."
  5. Keep the anti-metric banner and research insights. They're the emotional backbone of the experience.

What do you think? Anything you want to push back on or prioritize differently?
