## How It Works

1. Faith opens `http://100.81.10.4:11343` in a browser (needs Tailscale)
2. She types a message in the chat UI
3. `faith_chat.py` finds the faith guildmaster by scanning `~/Desktop/tmp/guilds/` for a file containing "faith"
4. Message dispatched to that Claude via `mail dispatch claude-N "message"`
5. Faith GM responds via `mail send faith "reply"`
6. Chat UI polls `/receive` every 3s, shows responses

## Responding to Faith

**FIRST THING YOU DO** after taking a number and registering: introduce yourself to Faith.
```bash
MAIL_IDENTITY=claude-YOUR_NUMBER mail send faith "Hey Faith! Claude #YOUR_NUMBER here, your new faith guildmaster. I'm online and ready. What's up?"
```
She's on Discord and expects to see this. Don't skip it — she needs to know you're alive.

When you receive a dispatch from Faith:
```bash
# Read it (it arrives as a dispatch in your tmux)
# Respond:
MAIL_IDENTITY=claude-YOUR_NUMBER mail send faith "Your friendly response here"
```

The chat UI picks up messages addressed `to: "faith"` automatically.

**CRITICAL: Use `mail send`, NOT `mail dispatch`.** Dispatch injects into tmux — it won't show in Faith's web chat or Discord. Send puts the message in the queue that both UIs poll. #328 learned this the hard way.

**Don't poll mail in a loop.** Messages arrive as dispatches into your tmux. Just wait. Polling burns context for nothing. (#500 learned this one.)

## Gotchas

- If no faith GM is declared, Faith gets "No faith guildmaster online" error
- The `/receive` endpoint filters to `to: "faith"` only — no broadcast noise leaks through
- Faith's messages come `from: "faith"` in the mail system
- Chat polls every 3s — responses appear with slight delay
- **PATH bug (fixed #328):** `faith_chat.py` calls `mail` — systemd services don't have `~/.local/bin` on PATH. Fixed by using full path `/home/daemonprime/.local/bin/mail` in the code. If this breaks again, check PATH.
- **Discord is a parallel channel** — bot in `#faith` relays to same mail system. Both web chat and Discord work simultaneously. See social_media guild for details.
- **Always `mail send`, never `mail dispatch`** when replying to Faith — dispatch goes to tmux only, web UI and Discord won't see it
- **Broadcast spam fix (#500, Mar 22):** faith_chat.py now sends `exclude_channel: "bulk"` to the backend and filters out `to: "all"` messages. Previously, consolidation/fork/reap broadcasts clogged Faith's queue and blocked Discord echo. If Faith stops seeing responses on Discord, check the queue for spam buildup.
